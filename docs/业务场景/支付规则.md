# 支付规则

## 支持的支付方式

| 方式 | 描述 | 限额 |
|------|------|------|
| 微信支付 | 扫码 / H5 | 单笔 ≤ 50000 |
| 支付宝 | 扫码 / 跳转 | 单笔 ≤ 50000 |
| 余额支付 | 账户余额 | 单笔 ≤ 充值余额 |

## 支付流程

1. 前端调创建支付 API → 获取支付链接/二维码
2. 用户完成支付 → 唤起支付回调
3. 后端接收异步通知 → 更新订单状态
4. 前端轮询订单状态 → 跳转结果页

## 关键规则

1. **幂等处理**：后端支付回调必须幂等，防止重复通知
2. **金额校验**：回调金额必须与订单金额一致，不一致拒绝
3. **签名校验**：所有回调需验签，防止伪造
4. **退款路径**：退款只能原路返回（微信→微信，支付宝→支付宝）
5. **超时处理**：支付二维码有效期 5 分钟，过期需重新生成

## 错误码

| 错误码 | 说明 | 处理方式 |
|--------|------|---------|
| PAY_TIMEOUT | 支付超时 | 引导重新下单 |
| BALANCE_INSUFFICIENT | 余额不足 | 提示其他支付方式 |
| AMOUNT_LIMIT | 超限额 | 分单或提额 |
| SIGN_ERROR | 签名错误 | 联系技术支持 |
