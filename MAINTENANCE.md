# 知识库维护手册 (Maintenance Guide)

本手册旨在指导人工或 Agent 如何维护本项目，确保知识库内容的准确性、时效性以及 MCP 服务的稳定性。

---

## 1. 目录结构说明

- `docs/`: 存放所有知识库 Markdown 原文件。
  - `业务场景/`: 业务流程、支付规则等文档。
  - `UI组件/`: 前端组件使用说明、API 文档。
  - `编码规范/`: Vue 规范、命名规范等。
- `mcp-server/`: MCP 服务核心代码。
  - `server.py`: 服务逻辑与 RAG 实现。
  - `.chroma/`: 向量数据库存储目录（自动生成）。
- `bin/`: 存放启动脚本 `knowledge-mcp`。

---

## 2. 内容维护 (人工/Agent)

### 2.1 新增/修改文档
- **存放位置**: 请根据分类将 `.md` 文件放入 `docs/` 下的子目录。
- **文档格式**: 
  - 必须使用标准 Markdown 格式。
  - **关键点**: 使用 `##` (二级标题) 进行内容分块。MCP Server 会根据二级标题切分知识片段，以提供更高精度的检索。
  - **建议**: 每个分块内容（二级标题下）保持在 1000 字以内，并包含完整的上下文描述。

### 2.2 示例代码
- 文档中的代码块应标明语言标签（如 ` ```vue ` 或 ` ```javascript `），这有助于 Agent 学习和生成代码。

---

## 3. 系统维护 (人工)

### 3.1 环境准备
- 确保 Python 3.10+ 环境。
- 安装依赖: `cd mcp-server && pip install -r requirements.txt`。

### 3.2 索引管理
- **自动同步**: 每次启动 MCP Server 时，系统会自动扫描 `docs/` 目录，并根据文件的修改时间 (mtime) 增量更新向量库。
- **手动刷新**: 如果在服务运行期间修改了文档，可以调用工具 `refresh_index` 或重启服务。
- **重置数据库**: 如果搜索质量异常，可手动删除 `mcp-server/.chroma/` 目录并重启服务，强制全量重新索引。

### 3.3 预览文档站
- 本项目集成 VitePress，可作为 Web 站点维护。
- 启动预览: `npm run docs:dev`。
- 构建发布: `npm run docs:build`。

---

## 4. Agent 维护指令 (面向 Agent)

如果你是一名正在维护本仓库的 Agent，请遵循以下 SOP：

1. **更新内容**: 修改 `docs/` 下的文件后，必须调用 `refresh_index` 工具确保向量库与文件同步。
2. **搜索质量**: 如果发现 `search_knowledge` 返回的内容不匹配，请检查对应文档的 `##` 标题是否合理，并尝试优化文档结构。
3. **安全性**: 严禁通过 `get_document` 读取 `docs/` 目录以外的任何系统文件。
4. **日志检查**: 运行 `knowledge-mcp` 时观察终端输出，确保 Embedding 过程无报错。

---

## 5. 故障排查

- **搜索不到内容**: 
  - 检查文档是否为 `.md` 结尾。
  - 检查 `KB_DIR` 环境变量是否正确指向了 `docs` 目录。
- **启动报错**: 
  - 检查端口是否被占用 (默认 8000)。
  - 检查是否缺少 `sentence_transformers` 或 `chromadb` 依赖。
- **Agent 无法连接**: 
  - `stdio` 模式下，检查命令路径是否绝对路径。
  - `http` 模式下，检查 `token` 是否匹配。
